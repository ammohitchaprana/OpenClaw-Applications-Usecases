## 🔒 Security Best Practices

Before connecting OpenClaw to **any** real account, follow these rules:

| Rule | Why |
|------|-----|
| **Never give OpenClaw root access** | Create a dedicated non-root user. Limit blast radius. |
| **Use dedicated accounts** | Separate Gmail/GitHub created just for OpenClaw. Not your primary accounts. |
| **Add human-approval gates** | For anything involving money, outbound emails, or file deletion, require explicit "yes." |
| **Start read-only** | Calendar reading before writing. Email summarizing before sending. |
| **Run in Docker** | Containerization limits damage if something goes wrong. |
| **Never hardcode API keys** | Use environment variables and secret management. |
| **Review all skills** | Check ClawHub's VirusTotal reports. Read source code before installing. |
| **Financial integrations: monitoring only** | Never give write access to financial accounts. |
| **Set up SOUL.md guardrails** | Define what the agent absolutely cannot do. |
| **Keep OpenClaw updated** | Security patches are frequent. Update regularly. |

> For comprehensive security guidance, see [CrowdStrike's analysis](https://www.crowdstrike.com/en-us/blog/what-security-teams-need-to-know-about-openclaw-ai-super-agent/) and the [official security docs](https://docs.openclaw.ai).

## 🏁 Getting Started

If you're new to OpenClaw, here's the fastest path to your first automation:

```bash
# 1. Install OpenClaw
npx openclaw@latest onboard

# 2. Run the onboarding wizard
openclaw onboard

# 3. Connect your messaging app (Telegram is easiest)
# Follow the wizard prompts

# 4. Install a skill
clawhub install morning-brief

# 5. Start with something simple
# Message your agent: "Give me a morning briefing every day at 7am"
```

**Recommended first automations** (pick one):
1. ☀️ Morning Brief — low risk, immediate value
2. 📧 Email Summary — read-only, saves hours
3. 📰 Reddit/YouTube Digest — fun and useful
